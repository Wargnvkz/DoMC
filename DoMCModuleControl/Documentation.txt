namespaces:
1. DoMCModuleControl - Основное пространство имен. В нем.
   а) AssemblyLoader - Загрузчик сборок. Нужен, потому что контроллер увидит сборки, которые находятся в памяти, а они загружаются по мере выполнения. Поэтому нужно загружать их руками
   б) IMainController - интерфейс описания главного контроллера. Все нужные функции для работы модуля и интерфейса. Через него осуществляется базовая загрузка управление модулями, интерфейсами, наблюдателем и командами
   в) MainController  - Реализация интерфейска главного контроллера
   г) Observer - наблюдатель. Позволяет подписаться на события уведомления. Все подписчики вызываются асинхронно. Ссылку на наблюдатель можно получить в MainController. Имя события обычно в виде <имя модуля>.<Область работы>.<результат/событие>
   д) ThrottledErrorNotifier - Надстройка над Observer, чтобы можно было ограничивать количество событий посылаемых через Observer. Например, если ошибка выполнения происходит раз в 1 секунду, но она некритичная и вы не хотите перегружать поток сообщений, то можно настроить передавать его, например, раз в минуту
2. DoMCModuleControl.Commands - базовые классы для описания и выполнения комманд
  а) CommandBase - абстрактный класс команды. Реализует базовый функционал для вызова функции в ревлизации класса.
  б) CommandBase.Event - список результатов действия/событий команды ()
  в) CommandInfo - параметры команды, которые хранятся в MainController и по нему он создает экземпляр новуй команды
  г) CommandModuleTypeAttribute - аттрибут для команды, говорящий к какому модулю относится команда, чтобы ее можно было вызвать по имени.
3. DoMCModuleControl.Logging - блок логирования
   а) BaseFilesLogger.cs - основаная система записи в файлы. Ему передаются названия модуля и текст, а он пишет лог в файл соответствующего каталога, разделяя файлы по датам
   б) BaseSystemLogger.cs - Предполагается, как система логирования в ОС, если BaseFilesLogger по какой-то причине не может записать лог в файл
   в) IBaseLogger.cs - интерфейс основного логгера добавление сообщения и регистрация системного логгера
   г) ILogger.cs - интерфейс для промежуточного класса для BaseFilesLogger для каждого модуля
   д) Logger.cs - экземпляр интерфеса ILogger
   е) LoggerLevel.cs - уровни логирования от Critical пишется всегда, Information - дополнительная информация, FullDetailedInformation - полная детальная информация
4. DoMCModuleControl.Modules
   а) ModuleBase.cs - абстрактный класс для реализации модуля. По нему MainController находит классы модулей
5. DoMCModuleControl.UI
   а) IMainUserInterface.cs - Главный интерфейс пользователя. По нему MainController опередляет интерфейс, который нужно открывать при загрузке


Парадигма: 
 вся передача данных идет в модули через команды, а от модулей либо через команды, если получить результат можно сразу, либо через наблюдатель.
 внутри модуля все необходимое получается от контролллера: логирование, наблюдатель, конфигурация

При создании MainController он просматривает все сборки на наличие реализаций IMainUserInterface и CommandBase и запоминает их. 
Интерфейс открывается автоматически
По имени в виде <имя класса модуля>.<имя класса команды> можно получить экземпляр класса команды. Либо класс команды должен быть внутри класса модуля, либо у него должен быть аттрибут CommandModuleTypeAttribute
MainController создает свой наблюдатель для подписки, которые можно получить от контроллера
Класс логирование для конкретного случая можно получить у контроллера
